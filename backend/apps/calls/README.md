# 通话系统 (Calls)

通话系统是视频交友应用的核心功能模块，负责管理WebRTC视频通话、语音通话、群组通话等功能。

## 🏗️ 系统架构

### 核心模型

1. **CallSession** - 通话会话
   - 通话基础信息（会话ID、类型、状态）
   - 参与者管理（发起者、参与者列表）
   - 通话设置（私密、录制、最大时长）
   - 统计信息（时长、质量评分）

2. **CallParticipant** - 通话参与者
   - 参与信息（角色、状态、连接信息）
   - 媒体控制（视频、音频、静音、摄像头）
   - 设备信息（设备类型、网络状态）
   - 参与统计（加入时间、总时长）

3. **CallInvitation** - 通话邀请
   - 邀请管理（邀请者、被邀请者）
   - 邀请状态（待处理、已接受、已拒绝）
   - 邀请消息和响应时间

4. **CallQuality** - 通话质量
   - 质量指标（视频、音频、网络质量）
   - 详细参数（比特率、丢包率、延迟、抖动）
   - 错误记录和统计

5. **CallRecording** - 通话录制
   - 录制管理（录制ID、状态、文件信息）
   - 录制设置（视频、音频、质量）
   - 权限控制（公开、下载权限）

## 📡 API接口

### 通话会话
```
GET/POST /api/v1/calls/              # 通话列表/创建
GET/PUT/DELETE /api/v1/calls/{id}/   # 通话详情/更新/删除
POST /api/v1/calls/{id}/start/       # 开始通话
POST /api/v1/calls/{id}/end/         # 结束通话
GET /api/v1/calls/stats/             # 通话统计
```

### 通话参与者
```
POST /api/v1/participants/           # 加入通话
```

### 通话邀请
```
POST/GET /api/v1/invitations/        # 发送/获取邀请
POST /api/v1/invitations/{id}/accept/ # 接受邀请
POST /api/v1/invitations/{id}/reject/ # 拒绝邀请
```

### 通话质量
```
POST /api/v1/quality/                # 提交通话质量数据
```

### 通话录制
```
POST/GET /api/v1/recordings/         # 开始/获取录制
```

### WebRTC信令
```
POST /api/v1/webrtc/offer/           # 处理Offer
POST /api/v1/webrtc/answer/          # 处理Answer
POST /api/v1/webrtc/ice-candidate/   # 处理ICE Candidate
```

### 其他接口
```
GET /api/v1/my-calls/                # 我的通话记录
POST /api/v1/action/                 # 通话操作
```

## 🎯 WebRTC实现

### 信令流程
1. **Offer阶段** - 发起方创建Offer
2. **Answer阶段** - 接收方创建Answer
3. **ICE阶段** - 交换ICE Candidate
4. **连接建立** - 建立P2P连接
5. **媒体传输** - 开始音视频传输

### 连接管理
- **STUN/TURN服务器** - NAT穿透和媒体中继
- **ICE连接检查** - 网络连接测试
- **连接状态监控** - 实时连接质量
- **自动重连** - 连接断开自动恢复

### 媒体控制
- **视频编码** - H.264/VP8/VP9支持
- **音频编码** - Opus/AAC支持
- **自适应比特率** - 根据网络调整
- **媒体约束** - 分辨率、帧率控制
