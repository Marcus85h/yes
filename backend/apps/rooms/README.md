# 房间系统 (Rooms)

房间系统是视频交友应用的核心模块，负责管理用户之间的视频通话房间、匹配和互动。

## 🏗️ 系统架构

### 核心模型

1. **Room** - 房间主模型
   - 房间基础信息（ID、名称、描述）
   - 房间配置（类型、最大人数、匹配条件）
   - 房间状态管理（等待、匹配中、已连接、已断开）
   - 统计信息（访客数、连接数、平均评分）

2. **RoomParticipant** - 房间参与者
   - 用户参与信息（角色、状态、连接时间）
   - 参与统计（总时间、评分）
   - 连接管理（连接、断开、离开）

3. **RoomMatch** - 房间匹配
   - 匹配请求管理
   - 匹配条件设置
   - 匹配状态跟踪

4. **RoomRating** - 房间评分
   - 用户间评分系统
   - 评分统计和计算

5. **RoomReport** - 房间举报
   - 举报管理
   - 处理流程

## 📡 API接口

### 房间管理
```
GET    /api/v1/rooms/              # 房间列表
POST   /api/v1/rooms/              # 创建房间
GET    /api/v1/rooms/{id}/         # 房间详情
PUT    /api/v1/rooms/{id}/         # 更新房间
DELETE /api/v1/rooms/{id}/         # 删除房间
POST   /api/v1/rooms/{id}/join/    # 加入房间
POST   /api/v1/rooms/{id}/leave/   # 离开房间
GET    /api/v1/rooms/stats/        # 房间统计
```

### 房间匹配
```
POST   /api/v1/match/              # 开始匹配
GET    /api/v1/match/              # 获取匹配状态
```

### 房间评分
```
POST   /api/v1/rating/             # 提交评分
```

### 房间举报
```
POST   /api/v1/report/             # 提交举报
```

### 其他接口
```
GET    /api/v1/my-rooms/           # 我的房间
POST   /api/v1/connect/{room_id}/  # 连接房间
POST   /api/v1/disconnect/{room_id}/ # 断开房间
```

## 🎯 匹配算法

### 匹配条件
1. **性别匹配** - 根据用户偏好匹配性别
2. **年龄匹配** - 年龄范围匹配
3. **地区匹配** - 地理位置偏好
4. **兴趣爱好** - 共同兴趣加分
5. **在线状态** - 在线用户优先
6. **认证状态** - 认证用户加分
7. **评分权重** - 高评分用户优先

### 匹配流程
1. 用户提交匹配请求
2. 系统查找符合条件的待匹配用户
3. 计算匹配分数
4. 选择最佳匹配
5. 创建匹配房间
6. 通知双方用户
